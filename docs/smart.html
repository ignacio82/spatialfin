How it works

Smart playback is a stack, not a checkbox.

SpatialFin now combines language preferences, track ranking, memory, and AI tooling so common playback decisions need fewer manual corrections.

Smart Audio And Subtitles

  • Ordered spoken languages: Users can add, search, and reorder the languages they understand.
  • OS-aware defaults: The first spoken language starts from the headset OS language before the user customizes it.
  • Original audio preference: SpatialFin can favor likely-original audio instead of defaulting to the first dub it finds.
  • Subtitle fallback: If selected audio is not in a spoken language, SpatialFin tries to enable an appropriate subtitle track automatically.
  • Subtitle ranking: When several subtitle tracks share the same language, SpatialFin now ranks candidates instead of blindly taking the first one.

Series-Level Memory

  • Audio memory: Switch one episode of a series to Japanese and future episodes can start there too.
  • Subtitle memory: SpatialFin stores the exact subtitle variant, not just the language, when you make a better track choice manually.
  • Correction-friendly behavior: Smart defaults are useful, but user overrides should win and stay won.

Voice And AI

  • Deterministic first: Common playback commands are handled directly for reliability.
  • On-device where available: SpatialFin includes an AICore / Gemini Nano path for compatible runtimes.
  • Cloud fallback when needed: Users can supply a Gemini API key for devices where AICore is absent.
  • Telemetry built in: Device status, AICore availability, and inference path details are visible for debugging.

Current reality

Galaxy XR still needs cloud fallback for Gemini features.

Current testing on Galaxy XR firmware shows that the system image does not expose com.google.android.aicore to third-party apps, so on-device Gemini features are currently unavailable there. SpatialFin’s voice stack still works through deterministic handling and optional cloud fallback.
